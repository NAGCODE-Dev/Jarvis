# Programming Knowledge Graph

- Programming logic -> enables algorithm design.
- Data structures -> shape time-space tradeoffs for algorithms.
- Algorithms -> run inside software architecture constraints.
- SOLID and design patterns -> help local maintainability, not absolute correctness.
- APIs -> connect bounded contexts and clients.
- Databases -> persist state; schema and query design influence system behavior.
- Distributed systems -> introduce latency, partial failure, retries, idempotency, and consistency tradeoffs.
- Docker and Linux -> define runtime packaging and operational surface.
- Git -> controls collaboration history and change safety.
- Security -> overlays every layer: input validation, authz, secrets, transport, dependency risk, logging.
- Testing -> validates logic, integration boundaries, and production assumptions.
- DevOps -> shortens feedback loops from code to deployment to observability.
- LLM systems -> combine model behavior, retrieval, tool use, context management, and evaluation.
- RAG -> joins vector retrieval and generation.
- Agents -> add planning, memory, tool invocation, and environment interaction on top of LLMs.
